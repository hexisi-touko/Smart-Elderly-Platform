## 二、分阶段实施计划

> [!IMPORTANT]
> 建议按 Phase 顺序推进。每个 Phase 内部的模块可并行。
> 每个 Phase 完成后都能形成**可演示的闭环**，方便向老师汇报。

## 四、开放问题

> [!WARNING]
> 以下问题需要你确认，会影响计划的具体实施方式。

1. **监护人登录**：监护人是否使用同一个 App？还是单独一个入口？我倾向于同一个 App 内做角色切换。
2. **用药提醒的推送方式**：uni-app 的本地通知即可，还是需要接入第三方推送（如极光推送）？
3. **地图功能**：SOS 定位和安全区域是否需要接入真实地图 SDK（如高德）？还是先用模拟数据？
4. **在线课程的媒体来源**：课程视频存在哪里？本地服务器 / OSS / 外链？
5. **优先级调整**：你觉得上面的 Phase 排序是否合理？有没有老师特别强调的模块需要提前？
